# 调研发现

## CDP 审计，移动端 433 x 937

- 当前文档高度 3367px，主界面按面板串行堆叠。
- 编辑器高 667px，预设区高 531px，结果区从 y=2118px 才开始。
- 视觉分割线宽或高为 4px，触摸目标不足。
- 上传后存在 43 个可聚焦控件，主流程缺乏层级。

## 设计约束

- 单一青绿色强调色，深色页面主题保持一致。
- 动效只用于抽屉和模式切换的状态反馈，支持 reduced motion。
- 画布为产品真实交互对象，不构造假预览。

## 重构后 CDP 移动端初验

- 390px 视口下，上传后的主工作区高度 1450px，但画布与固定“调整 / 导出”操作在当前视口可达，低频功能未进入主页面。
- 调整后底部面板高度为 338px，符合 36dvh 初始状态。
- 横向分割线命中区为 350 x 36px，纵向为 36 x 350px，已解决原先 4px 触控目标问题。
- CDP 的首次按钮点击发生在 Vite HMR 完成前；通过 DOM click 确认状态可正常进入调整模式。后续验证以刷新后稳定状态为准。
- 在调整模式增加横线后，分割线数量从 2 变为 3，主操作从“导出 4 张”更新为“导出 6 张”。
- 点击导出会等待切片生成完成，再展示“6 个切片已准备好”和“下载 6 张”的确认抽屉。

## 功能回归修复与最终 CDP 证据

- 根因：`AppHeader.vue` 依赖旧 `App.vue` 的全局布局样式；重构后 header 高达 1284px，画布被推到 y=1334。已把 header 样式迁回组件，PC header 现为 64px，画布从 y=100 开始。
- 新 host 已验证：`http://image-sli.localhost:5188/`。
- PC：1440px 视口中画布为 928px 宽、836px 高，512px 原图按原尺寸置中，不溢出。
- 移动端：390px 视口中 header 为 54px、画布为 694px 高，固定栏只保留“调整 / 导出”；分割线热区为 36px。
- 上传后隐藏 file input 仍挂载；实测追加第二张图片，队列从 1 变为 2，总导出从 4 变为 8。
- 导出抽屉实测显示两个图片组、8 个切片、逐图下载和总下载入口。

## 代码落点

- 当前 `App.vue` 同时负责所有视图状态与业务流程，适合作为工作台状态编排层。
- `SliceEditor.vue` 将收敛为只负责画布叠线、选择和拖动，设置控件上移到工作台抽屉。
- `SlicePresetPanel.vue` 保留业务事件契约，仅迁移到全屏预设层，不再作为主页面区块。
