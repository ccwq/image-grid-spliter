## ADDED Requirements

### Requirement: Chromium 同目录批量导出

系统 SHALL 在浏览器支持 File System Access API 时，让用户通过“下载全部”首次选择一个具备读写权限的目录，并将本次所有切片直接写入该目录，不对每张切片触发传统下载。

#### Scenario: 首次选择目录后写入全部切片
- **WHEN** 支持的浏览器中用户点击“下载全部”且尚未存在可写目录句柄
- **THEN** 系统在该点击上下文中显示目录选择器
- **AND** 用户确认后将所有切片直接写入所选目录

#### Scenario: 已授权目录直接复用
- **WHEN** 用户点击“下载全部”且已保存目录句柄的读写权限为 `granted`
- **THEN** 系统不再次显示目录选择器
- **AND** 将全部切片直接写入该目录

### Requirement: 目录句柄复用与更换

系统 MUST 将用户选定的目录句柄保存到 IndexedDB，并在后续会话尝试复用；当权限为 `prompt` 时仅能在用户主动导出时请求授权，且系统 SHALL 提供更换导出目录的操作。

#### Scenario: 恢复的句柄要求重新授权
- **WHEN** 系统从 IndexedDB 恢复目录句柄且其权限为 `prompt`
- **THEN** 用户点击“下载全部”后系统请求读写授权
- **AND** 授权成功时复用该目录，失败时走传统逐张下载

#### Scenario: 用户更换目录
- **WHEN** 支持的浏览器中用户选择“更换导出目录”并确认新目录
- **THEN** 系统保存新目录句柄
- **AND** 后续目录直写使用新目录

### Requirement: 文件不覆盖

系统 MUST 不覆盖目标目录中的已有文件；重名切片 SHALL 在扩展名前使用 `-1`、`-2` 等递增后缀生成可用名称。

#### Scenario: 存在同名切片
- **WHEN** 目标目录已有 `photo-r1c1.png`
- **THEN** 系统写入 `photo-r1c1-1.png` 或下一个未占用的递增后缀名称
- **AND** 原文件内容保持不变

### Requirement: 兼容降级和自动下载

系统 SHALL 在浏览器不支持目录直写、用户拒绝/取消权限或没有有效目录句柄时保留既有传统逐张下载；系统 MUST NOT 提供 ZIP 导出。自动下载仅在已有有效目录写入权限时进行目录直写。

#### Scenario: 不支持目录直写的浏览器
- **WHEN** 浏览器不提供 `showDirectoryPicker`
- **THEN** 系统隐藏更换目录控制并显示传统下载说明
- **AND** “下载全部”按现有逐张下载方式执行

#### Scenario: 自动下载没有可用目录
- **WHEN** 自动下载开启但没有权限为 `granted` 的目录句柄
- **THEN** 系统不得弹出目录选择器
- **AND** 按既有逐张下载方式下载切片

### Requirement: 部分失败的显式恢复

系统 MUST 在目录直写任一文件失败时停止后续写入，展示成功和未完成数量；系统 SHALL 仅在用户明确操作后才将未完成文件按传统逐张下载重试。

#### Scenario: 中途写入失败
- **WHEN** 某个切片写入目录失败且此前已有切片写入成功
- **THEN** 系统停止目录写入并报告成功数量和未完成数量
- **AND** 不会自动把未完成切片下载到浏览器默认目录

#### Scenario: 用户重试未完成切片
- **WHEN** 用户在部分失败结果中明确选择传统下载重试
- **THEN** 系统仅触发未完成切片的传统逐张下载
