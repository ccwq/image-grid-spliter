## ADDED Requirements

### Requirement: 紧凑响应式工作区

系统 SHALL 在桌面端将控制与上传/结果分列展示，在移动端使用单列折叠控制，并保持所有既有功能入口可达。

#### Scenario: 桌面端查看结果
- **WHEN** 视口宽度至少为 960px 且用户已上传图片
- **THEN** 网格、裁切和导出控制显示在紧凑左侧栏
- **AND** 原图、切片宫格和下载/预览操作显示在右侧主区域

#### Scenario: 移动端编辑与下载
- **WHEN** 视口宽度小于 960px
- **THEN** 页面使用单列，控制区可折叠
- **AND** 用户无需进入新页面即可上传、编辑、预览和下载

### Requirement: 分阶段导出进度

系统 MUST 在导出任务期间显示 fixed 顶部进度，明确显示生成、目录保存或传统下载触发阶段的当前文件数和总文件数。

#### Scenario: 生成后目录保存
- **WHEN** 用户导出多个切片且目录直写可用
- **THEN** 顶部依次显示生成和保存阶段
- **AND** 每个阶段显示递增的 current / total

#### Scenario: 传统下载降级
- **WHEN** 导出使用传统逐张下载
- **THEN** 顶部显示“正在触发下载 current / total”
- **AND** 不将其表述为已保存到磁盘

### Requirement: 导出完成报告

系统 SHALL 在导出完成后将顶部进度切换为报告。成功报告五秒后自动收起，部分失败报告保持显示并提供恢复操作。

#### Scenario: 目录导出成功
- **WHEN** 所有目录写入完成
- **THEN** 报告已保存总数、重命名数和失败数
- **AND** 五秒后自动收起，除非用户提前关闭

#### Scenario: 部分写入失败
- **WHEN** 目录写入中途失败
- **THEN** 报告已保存和未完成数量并保持显示
- **AND** 用户可选择传统下载重试未完成项

### Requirement: 功能回归保护

系统 MUST 保留既有上传、裁切、格式、目录导出、传统下载、预览、宫格、PhotoSwipe、本地化和 PWA 能力。

#### Scenario: 布局重构后的多图主流程
- **WHEN** 用户追加多张图片、编辑网格并选择任一导出方式
- **THEN** 每张图片的切片、预览和下载操作仍可用
- **AND** 布局改变不得移除或隐藏该流程所需入口
