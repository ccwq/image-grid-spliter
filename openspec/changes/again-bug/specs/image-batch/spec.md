## MODIFIED Requirements
### Requirement: 多文件上传与拖拽队列
系统 SHALL 支持通过文件选择或拖拽一次添加多张图片，按加入顺序维护待处理队列；非图片文件需被忽略并给出提示，不得覆盖已有队列内容。系统 MUST 避免同一文件在一次或多次重复触发的拖拽/选择事件中重复入队，包括移动端触摸拖拽产生的多次 drop 事件。

#### Scenario: 文件选择追加多图
- **WHEN** 用户在文件选择器中一次选择多张图片或分批选择追加
- **THEN** 队列按选择顺序包含所有有效图片
- **AND** 非图片被过滤并提示，已有队列内容保留

#### Scenario: 拖拽多图进入队列
- **WHEN** 用户拖拽多张图片到上传区域
- **THEN** 系统批量接收有效图片并保持顺序
- **AND** 不会清空已有队列，可继续追加

#### Scenario: 重复触发的拖拽不重复入队
- **WHEN** 同一文件被拖拽到上传区域且事件被浏览器/框架多次触发（例如移动端 drop 冒泡至全局监听）
- **THEN** 队列仅包含该文件的一条记录，顺序以首次接收为准
- **AND** 不会出现重复切片或重复下载的额外条目，错误状态保持清晰
