## ADDED Requirements

### Requirement: 预览中的优先保存

系统 SHALL 在切片预览顶部提供“保存”动作。该动作 MUST 优先将当前切片写入已授权的导出目录；当目录不可用、未授权或写入失败时，系统 MUST 自动以传统浏览器下载完成该文件，并明确说明降级结果。

#### Scenario: 已授权目录保存成功
- **WHEN** 用户在预览中点击“保存”且已有可写目录
- **THEN** 系统将当前切片 Blob 直接写入该目录
- **AND** 按钮与 Toast 显示“已保存”反馈

#### Scenario: 无可写目录时保存
- **WHEN** 用户在预览中点击“保存”但没有可写目录
- **THEN** 系统使用传统下载保存当前切片
- **AND** Toast 明确说明已降级为浏览器下载

### Requirement: 预览动作分离与反馈

系统 SHALL 在预览顶部并列提供“保存”和“下载”按钮，并提供“另存为”入口。系统 MUST 在操作进行中显示按钮内状态，并在完成、取消、失败或降级时提供用户可见反馈。

#### Scenario: 用户选择传统下载
- **WHEN** 用户在预览中点击“下载”
- **THEN** 系统只触发传统 `<a download>` 下载
- **AND** 不尝试请求目录权限

#### Scenario: 用户另存为
- **WHEN** 用户在预览中点击“另存为”且浏览器支持文件保存选择器
- **THEN** 系统打开文件保存对话框并将当前切片写入用户选择的位置
- **AND** 完成或取消后显示对应反馈
