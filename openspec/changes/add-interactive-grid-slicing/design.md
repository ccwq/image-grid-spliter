## Context

图片的实际格子宽高经常并不均等，且格子边线可能包含需要裁掉的 gap。现有算法只接收 rows/cols，不能表达这些信息。

## Decisions

- 分割线以 0 到 1 的归一化位置保存，第一张图预览作为编辑基准；每张批量图片都按自身自然尺寸映射。
- 所有分割线都是贯穿式横线或竖线，输出保持矩形网格。
- padding 表示一条边线的总擦除宽度；内部线由相邻两块各裁一半。外边缘开关开启时，外边各裁一半 padding。
- 预设格式为版本化 JSON，不携带图片二进制。
- 画布始终是上传后页面的中心内容。移动端固定操作仅保留“调整”和“导出”，其余操作放在调整面板或预设层中。
- 分割线视觉宽度保持细线，但命中区最小为 36px；仅在调整模式响应拖动。
- 任何上传入口复用同一个常驻 file input，避免上传后因 DOM 卸载而无法追加图片。
- 导出采用确认抽屉，保留每张输入图的切片预览和逐图下载。

## Risks

- padding 可能使很窄的块无可用内容；算法将忽略零尺寸块，UI 显示有效切片数。
- 预览渲染尺寸和原图尺寸不同；指针坐标始终转为归一化坐标后再传递。
