## ADDED Requirements
### Requirement: 模块化的 App Shell
App shell SHALL 将页面拆分为职责清晰的子组件与组合式逻辑，避免核心交互集中在单一文件。
- 子组件至少覆盖：头部信息（品牌/版本/语言切换）、网格预设与自定义行列、导出设置（格式/质量）、上传/拖拽入口、结果预览与下载区域。
- 共享逻辑（如本地化消息、网格状态、上传与裁切流程、下载触发）应通过组合式 API 或独立工具暴露，子组件仅接收所需 props 与 emits。
- 组件/组合式需包含关键注释，解释状态流转、异步处理与事件触发链路，保持类型完整（props/emits/state 均有显式类型）。

#### Scenario: 页面结构按职责拆分
- **WHEN** 渲染 `App` 页面
- **THEN** 页面使用独立子组件组合头部、网格、导出、上传、结果等区域，而非单文件堆叠模板/逻辑

#### Scenario: 共享逻辑可复用
- **WHEN** 子组件需要使用网格选择、上传/裁切、下载或本地化逻辑
- **THEN** 相关逻辑通过组合式/工具暴露，子组件仅依赖显式的 props/emits，核心状态与副作用集中管理

#### Scenario: 可读性友好
- **WHEN** 阅读子组件与组合式
- **THEN** 关键状态与异步流程具备简洁注释与类型标注，帮助理解数据流、错误处理与交互入口
