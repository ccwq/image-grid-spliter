# UI 单行工具条验收工作流

## 触发器

任何涉及响应式布局、工具条、表单控件、折叠区或按钮排列的 UI 改动完成后触发。该工作流适用于整个项目，不限于自定义网格和导出设置。

## 目标

确保图像网格裁切器中的紧凑工具行在桌面与移动端保持可操作、信息层级清晰，并且不会因响应式规则折行。

## 已确认布局契约

- 自定义网格是预设区下方的一条独立操作行。
- 此行从左到右包含：列输入、行输入、右侧的“自定义网格”标签和“使用自定义网格”按钮。
- 上述四个元素必须处于同一视觉行，禁止任何元素换行。
- 行使用 `display: flex`、`flex-direction: row`、`flex-wrap: nowrap`；不得以横向滚动作为兜底。
- 间距紧凑且有分组关系：列/行构成左侧参数组，标签/按钮构成右侧操作组。

## 验收证据

- 通过 CDP 在桌面和 375px 移动视口截图。
- 读取该行的计算样式，确认 `flex-direction: row`、`flex-wrap: nowrap`。
- 检查子元素边界均未越过容器右边界。

## 执行步骤

1. 在桌面视口和 375px 移动视口打开受影响页面。
2. 截图检查信息层级、组间距、按钮可读性和输入框对齐。
3. 对每个要求单行的工具条读取计算样式，确认 `flex-direction: row` 与 `flex-wrap: nowrap`。
4. 比较父容器与子元素边界，确认没有子元素溢出、横向滚动或被裁切。
5. 运行与改动范围相称的类型检查和测试。

## 完成标准

- 所有被标为单行的区域，在 375px 和桌面视口均为一行。
- 所有控件仍可点击或输入，且无重叠、裁切、横向滚动。
- 视觉上参数与动作有清晰分组，组内间距小于组间间距。
- CDP 截图与计算样式作为验收证据保留在临时截图目录。
- 类型检查和相关测试通过。
