# 多分辨率 UI 审查与修复

## 目标

在不改变裁切和下载流程的前提下，改善移动与桌面端的视觉层级、按钮密度、PWA 颜色连续性，以及已选中/未选中状态的辨识度。

## 成功标准

- 在 320、390、768、1024、1440px 视口中无横向溢出和文字/按钮变形。
- 选图后上传状态紧凑，主次操作清晰。
- 网格预设、边线擦除、导出格式和悬浮操作组的默认、选中、未选中、禁用、焦点状态可区分。
- PWA 页面、状态栏和启动背景使用连续深青黑色。
- 裁切、预览、下载、Drawer 与键盘可访问性不回归。

## 最大轮次

1. 多分辨率与状态基线审查。
2. 主代理实施针对性 CSS/组件修正。
3. 独立复审与回归检查。
4. 仅修复复审发现的问题并完成最终验证。
